# REST API DNSO

Базовый URL: `http://<host>:<port>` (по умолчанию `http://127.0.0.1:8080`)

Формат ответа: `application/json; charset=utf-8`

---

## Зоны

### GET /api/zones

Получить список всех зон.

**Пример ответа:**

```json
[
    {
        "Id": 1,
        "Name": "example.com.",
        "TTL": 300
    },
    {
        "Id": 2,
        "Name": "v95.",
        "TTL": 600
    }
]
```

---

### GET /api/zones/{name}

Получить зону по имени.

**Параметры пути:**

- `name` — имя зоны (FQDN, точка в конце опциональна)

**Пример ответа:**

```json
{
    "Id": 1,
    "Name": "example.com.",
    "TTL": 300
}
```

**Коды ошибок:**

- `404` — зона не найдена

---

### POST /api/zones

Создать новую зону.

**Тело запроса:**

```json
{
    "name": "example.com.",
    "ttl": 300
}
```

**Поля:**

- `name` (string, обязательное) — имя зоны. Если не заканчивается на `.`, точка добавляется автоматически.
- `ttl` (int, опциональное) — TTL по умолчанию. Если не указан или ≤ 0, устанавливается `300`.

**Пример ответа (201 Created):**

```json
{
    "Id": 1,
    "Name": "example.com.",
    "TTL": 300
}
```

**Коды ошибок:**

- `400` — некорректный JSON или пустое имя
- `409` — зона с таким именем уже существует

---

### PUT /api/zones/{name}

Обновить TTL зоны.

**Параметры пути:**

- `name` — имя зоны (FQDN, точка в конце опциональна)

**Тело запроса:**

```json
{
    "ttl": 600
}
```

**Поля:**

- `ttl` (int, опциональное) — новый TTL. Если не указан или ≤ 0, устанавливается `300`.

**Пример ответа:**

```json
{
    "Id": 1,
    "Name": "example.com.",
    "TTL": 600
}
```

**Коды ошибок:**

- `500` — внутренняя ошибка сервера

---

### DELETE /api/zones/{name}

Удалить зону.

**Параметры пути:**

- `name` — имя зоны (FQDN, точка в конце опциональна)

**Пример ответа:** `204 No Content`

**Коды ошибок:**

- `500` — внутренняя ошибка сервера

---

## Записи

### GET /api/zones/{name}/records

Получить список всех записей в зоне.

**Параметры пути:**

- `name` — имя зоны (FQDN, точка в конце опциональна)

**Пример ответа:**

```json
[
    {
        "Id": 1,
        "ZoneId": 1,
        "Domain": "www.example.com.",
        "Type": "A",
        "Rdata": "192.168.1.10",
        "TTL": 300
    },
    {
        "Id": 2,
        "ZoneId": 1,
        "Domain": "example.com.",
        "Type": "MX",
        "Rdata": "10 mail.example.com.",
        "TTL": 300
    }
]
```

**Коды ошибок:**

- `404` — зона не найдена
- `500` — внутренняя ошибка сервера

---

### POST /api/zones/{name}/records

Создать новую запись в зоне.

**Параметры пути:**

- `name` — имя зоны (FQDN, точка в конце опциональна)

**Тело запроса:**

```json
{
    "domain": "www.example.com.",
    "type": "A",
    "rdata": "192.168.1.10",
    "ttl": 300
}
```

**Поля:**

- `domain` (string, обязательное) — полное имя домена
- `type` (string, обязательное) — тип записи. Поддерживаемые значения: `A`, `AAAA`, `CNAME`, `MX`, `TXT`, `NS`, `SOA`. Регистр не важен.
- `rdata` (string, обязательное) — значение записи
- `ttl` (int, опциональное) — TTL записи. Если не указан или ≤ 0, используется TTL зоны.

**Пример ответа (201 Created):**

```json
{
    "Id": 1,
    "ZoneId": 1,
    "Domain": "www.example.com.",
    "Type": "A",
    "Rdata": "192.168.1.10",
    "TTL": 300
}
```

**Коды ошибок:**

- `400` — некорректный JSON или пустые обязательные поля
- `404` — зона не найдена
- `500` — внутренняя ошибка сервера

---

### PUT /api/records/{id}

Обновить существующую запись.

**Параметры пути:**

- `id` — числовой идентификатор записи

**Тело запроса:**

```json
{
    "zone_id": 1,
    "domain": "www.example.com.",
    "type": "A",
    "rdata": "10.0.0.1",
    "ttl": 600
}
```

**Поля:**

- `zone_id` (int, обязательное) — ID зоны
- `domain` (string, обязательное) — полное имя домена
- `type` (string, обязательное) — тип записи
- `rdata` (string, обязательное) — значение записи
- `ttl` (int, опциональное) — TTL. Если не указан или ≤ 0, устанавливается `300`.

**Пример ответа:**

```json
{
    "Id": 1,
    "ZoneId": 1,
    "Domain": "www.example.com.",
    "Type": "A",
    "Rdata": "10.0.0.1",
    "TTL": 600
}
```

**Коды ошибок:**

- `400` — некорректный JSON, пустые поля или невалидный ID
- `500` — внутренняя ошибка сервера

---

### DELETE /api/records/{id}

Удалить запись.

**Параметры пути:**

- `id` — числовой идентификатор записи

**Пример ответа:** `204 No Content`

**Коды ошибок:**

- `400` — невалидный ID
- `500` — внутренняя ошибка сервера

---

## Коды ошибок (общие)

| Код | Описание                          |
|-----|-----------------------------------|
| 200 | Успешный запрос                   |
| 201 | Ресурс создан                     |
| 204 | Ресурс удалён (нет тела ответа)   |
| 400 | Некорректный запрос (JSON, параметры) |
| 404 | Ресурс не найден                  |
| 409 | Конфликт (дубликат зоны)          |
| 500 | Внутренняя ошибка сервера         |

---

## Примеры использования с curl

```bash
# Создать зону
curl -X POST http://127.0.0.1:8080/api/zones \
  -H 'Content-Type: application/json' \
  -d '{"name": "example.com.", "ttl": 300}'

# Список зон
curl http://127.0.0.1:8080/api/zones

# Создать A-запись
curl -X POST http://127.0.0.1:8080/api/zones/example.com./records \
  -H 'Content-Type: application/json' \
  -d '{"domain": "www.example.com.", "type": "A", "rdata": "192.168.1.10"}'

# Список записей зоны
curl http://127.0.0.1:8080/api/zones/example.com./records

# Обновить запись
curl -X PUT http://127.0.0.1:8080/api/records/1 \
  -H 'Content-Type: application/json' \
  -d '{"zone_id": 1, "domain": "www.example.com.", "type": "A", "rdata": "10.0.0.1", "ttl": 600}'

# Удалить запись
curl -X DELETE http://127.0.0.1:8080/api/records/1

# Удалить зону
curl -X DELETE http://127.0.0.1:8080/api/zones/example.com.
